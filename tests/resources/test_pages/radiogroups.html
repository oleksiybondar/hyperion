RadioGroups (etalon)

This page contains radio group variants for Hyperion component tests:
1) wrapper + separate input/label
2) label wraps input
3) sibling input and label (no item wrapper)

Scenario 1: Wrapper items

Each item has a wrapper element. Input and label are separate. Labels are bound using for.

Suggested modeling: items = wrapper nodes; input = input[type=radio]; label = label

Scenario 2: Label wraps input

The label is the item root and contains the input. Clicking the label toggles the input.

Suggested modeling: items = label; input = input[type=radio]; label may be . (self) if supported.

Scenario 3: Sibling input and label (no wrapper)

Inputs and labels are siblings. Labels are bound using for. There is no per-item wrapper element.

Suggested modeling: items = input[type=radio]; input = . (self); label = following-sibling label (XPath) or label[for=...] strategy.
